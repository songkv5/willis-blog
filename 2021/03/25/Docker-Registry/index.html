Docker-registry

2021-03-25 | 阅读

Docker Registry 是一个无状态的服务端应用，可以帮我们存储镜像、在分布式系统中分发镜像。docker registry类似于我们maven私服nexus，我们可以将构建好的镜像推送到registry，也可以从registry将镜像拉下来。

Registry的用途

  • 控制我们的镜像保存在哪里
  • 控制镜像分发
  • 将镜像存储和分发应用到我们的内部开发工作流中。

将镜像推送到registry中

命令格式：docker push 镜像名【此时的镜像名称的格式是：registry地址/镜像名称】
参考官方文档：Understanding image naming

将本机景象标记到远程并打标签:
docker tag IMAGE[:TAG] [REGISTRY_HOST[:REGISTRY_PORT]/]REPOSITORY[:TAG]

查看registry中的镜像

访问地址：http://registry-path/v2/\_catalog
访问地址: http://192.168.99.118:5000/v2/镜像名称/tags/list, 可以查看镜像版本
