Swarm是什么

Docker Swarm是docker公司2014年出品的基于docker的集群管理调度工具,是docker内嵌的集群管理和编排工具。官方文档地址：https://docs.docker.com/engine/swarm/

swarm 的使用

官方文档

一、关键概念

  1. 节点（Node）：结点指的是加入到swarm中的一个Docker实例。结点分为管理者节点（Manager Node）和工作节点（Worker Node）。当发布一个应用到swarm，需要提交一个服务定义到管理者结点，管理者几点负责分发任务到工作结点。除此之外，管理者节点还负责管理集群，选举leader组织编排任务。工作结点接收和执行管理者节点下发的任务。

    管理者结点任务:
    1.维持集群状态
    2.调度服务(scheduling services)
    3.提供swarm模式的http接口

    需要注意的是，如果只有一个管理者，当管理者结点宕机，服务想要继续运行，需要创建一个新的集群来恢复。因此，为了提高容错性，docker建议我们，根据具体的高可用需求，配置响应的奇数个数的管理者结点。
    1.一个拥有3个管理者结点的swarm集群可以容忍1个管理者结点失败
    2.一个拥有2个管理者结点的swarm集群可以容忍2个管理者结点失败
    3.一个拥有N个管理者结点的swarm集群可以容忍(N-1)/个管理者结点失败

    工作结点
    工作结点也是Docker引擎，但是他唯一的目的就是运行容器。他不参与任务编排，也不提供swarm模式的http接口。

  2. 服务（Service）：服务是对要执行任务的定义，是swarm系统的核心结构，是用户与swarm沟通的根本。创建一个服务时，需要指定要使用哪个镜像、要在容器中执行哪些命令；当要运行多个副本是，还可以指定要运行的副本数量。

  3. 任务：一个任务附带一个Docker容器和要在容器中执行的命令。

  4. 负载均衡：Swarm manager使用内部负载均衡（internal load balancing）来分发请求

四、Docker Machine

  • 什么是Docker Machine，什么时候要用到Docker Machine？
    传送门
    官网释义：Docker Machine是一个可以让你在虚拟主机上安装Docker 引擎的工具.

为啥要用docker Machine

以下说明来自官网理解:
docker machine可以在各种linux平台上提供多种远程docker主机. 除此之外, Docker Machine允许我们在老版本的Windows或Mac系统上运行Docker. Docker Machine有以下两种最广泛的用途:

  1. 我有一个旧版本的桌面版系统(如Mac或Windows),而且我想运行Docker

    如果你主要的工作环境是一个老版本的桌面系统,而且不满足新版本的docker-destop(Mac或Windows)应用,那么就需要依靠Docker Machine在本地运行docker 引擎.
  2. 想在远程系统上提供docker主机

    Docker 引擎一般在Linxu系统上运行. 如果你有一台linux机器,而且你想执行Docker命令,所有你要做的事就是下载并安装docker引擎(Engine). 然而,如果你想要一个更高效的方式,在网络上提供多个Docker主机,比如在云上,或者本地,就需要使用Docker machine.不管你的系统是Mac,linux,或Windows,都可以安装Docker Machine并使用docker-machine命令来提供并管理大量的Docker主机. 这回自动创建主机,并在上面安装docker引擎,并配置Docker 客户端. 每个被管理的machine(即刚才创建的主机)都是docker主机和docker clients的组合.

docker engine 和docker machine的区别

Docker Engine是一个提供并管理「安装了Docker引擎的主机」的「工具」.最典型的,就是你可以在你本级系统安装Docker Machine. Docker Machine拥有它自己的命令行客户端「docker-machine」和Docker引擎客户端「docker」. 你可以使用Docker Machine在多个虚拟机上安装Docker engine. 这些虚拟机可以是本地的(当你使用Docker Machine在windows或者Mac上的VirtualBos上安装Docker engine时)或者远程的(当你使用Docker Machine在云平台上提供Docker化的主机时).这些docker化的主机(Dockerized hosts)可以被看作是被管理的“机器”(machines)
