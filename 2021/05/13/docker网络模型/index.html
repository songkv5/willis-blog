docker网络模型

一、理论基础

  • 设计标准
    CNM: 容器网络模型(Container network model), 是容器网络的设计标准,一套规范. 它规定了docker网络架构的基础组成要素.
    1. 沙盒(SandBox): 一个独立的网络栈,或者网络命名空间。其中包括以太网接口、端口、路由表以及 DNS 配置
    2. 终端(End Point): 虚拟网络接口。就像普通网络接口一样，负责接收网络连接,如一个veth对. 一个沙盒可能包含多个Endpoint*。
    3. 网络(Network): 802.1d 网桥（类似大家熟知的交换机）的「软件实现」.网络就是需要交互的终端的集合，并且终端之间相互独立。这个模块的作用就是保证容器之间的通信.

二、Docker具体网络实现:Libnetwork

Libnetwork实现了CNM标准的3个组件. Libnetwork是开源的，采用 Go 语言编写，它跨平台（Linux 以及 Windows），并且被 Docker 所使用。

三、网络驱动

网络连通性和隔离性是由驱动来处理的，驱动层实际创建网络对象也是如此，其关系如下图所示。

Docker 封装了若干内置驱动，通常被称作原生驱动或者本地驱动。
在 Linux 上包括 Bridge、Overlay 以及 Macvlan，在 Windows 上包括 NAT、Overlay、Transport 以及 L2 Bridge。
为了满足复杂且不固定的环境需求，Libnetwork 支持同时激活多个网络驱动。这意味着 Docker 环境可以支持一个庞大的异构网络。

四、Docker网络应用

在安装了docker的主机上,查看docker的网络列表

查看docker网络

  • bridge网路: Docker的默认网络驱动,如果我们在创建网络时没有指定驱动类型,那么就会创建bridge网络. bridge网络经常用在跑在单机容器的应用.
    默认的container都是连接到桥接(bridge)网络, 桥接网络网络会映射到docker主机内核的网络
  • host网络: 为单机容器设计,移除了容器与docker主机之间的网络隔离限制,直接使用主机的网络.
  • overlay网络: 将多个docker daemon网络连接到一起,并让swarm中的服务能够相互沟通.也可以让两个独立运行的容器沟通,只要两个container都运行与overlay网络.

  • ingress: 是一个特殊的overlay网络, 用于服务节点之间的「负载均衡」.当然和swarm节点在发布端口上接收到请求时,会将请求提交给一个IPVS模块. IPVS模块跟踪该服务的所有ip地址,选择其中一个,并通过ingress网络找到他. swarm初始化或者加入swarm集群的时候会自动创建ingress 网络.

网络负载均衡

swarm两种网络负载均衡: vip与dnsrr.
默认情况下, 发布了端口的swarm服务使用路由网格. 当我们连接到任意一个swarm节点暴露的端口上,我们的请求就会被重定向到一个运行着该服务的work节点上.
实际上, 在swarm服务中, Docker自己也扮演了负载均衡器的角色, 使用路由网格的服务都运行在vip(Virtual ip)模式. 也可以绕过vip模式,使用dnsrr(DNS round robin)模式,通过指定 –endpoint-mode dnsrr选项即可.

  • DNSRR 负载均衡
    优点: Docker 版本大于1.10即原生支持容器集群内部DNS的服务发现。
    缺点: 由于DNS TTL生效时间的存在，解析的结果不能做到实时，即使TTL设置为0，某些应用或者方法库会缓存DNS解析的结果，导致会解析到已经失效的地址上.

  • VIP 负载均衡
    在这种方法中，每个服务都有一个 IP 地址，并且该 IP 地址映射到与该服务关联的多个容器的 IP 地址。在这种情况下，与服务关联的服务 IP 不会改变，即使与该服务关联的容器死亡并重新启动。
    使用VIP负载均衡,最终会有一个虚拟IP与容器实际的IP地址列表的映射.
    虚拟ip使用linux内核的iptable和ipvs负载均衡到多个容器.iptables 实现防火墙规则，IPVS 实现负载均衡.

    通过执行docker exec -it CONTAINER_ID /bin/bash,进入容器终端tty, 通过ping demo_ws_sk,即可查看服务的虚拟IP.

    docker内置的DNS server会将服务名解析成一个虚拟ip(这里是10.0.1.2),解析道vip后,在内核的iptable和ipvs模块,有虚拟IP 10.0.1.2与实际节点ip的映射关系,就可以由内核完成负载均衡.

注: 有的地方将dnsrr和vip也统称为内部负载均衡

  • 路由网格(Routing mesh)

    路由网格是docker swarm默认的网络,swarm服务的所有节点都会在ingress网络上架一份.
    routing mesh依赖ingress网络实现,而且每个docker实例只允许有一个ingress网络.创建新的ingress网络可以在网络创建命令中使用–ingress选项指定.

Swarm 模式下，docker会创建一个默认的overlay网络—ingress network。Docker也会为每个worker节点创建一个特殊的net namespace（sandbox）– ingress_sbox。ingress_sbox有两个endpoint，一个用于连接ingress network，另一个用于连接local bridge network docker_gwbridge。

通过主机ip+端口进行访问, 该节点iptable模块将端口对应的sanbox的ip地址, 进如ingress网络空间后, 有sanbox里实现的iptable和ipvs进行路由转发.

集合上面创建的servicedemo_ws_sk, 再创建一个只有一个副本的service,且不指定网络,使用命令docker service create –replicas 1 –name demo_ws -p 8081:8080 –network demo-1
