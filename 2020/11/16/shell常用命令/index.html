查看系统内存

top命令，在子界面输入大写的E，上面可以查看总内存；多次按可以切换不同的单位

按小写的e，在下面可以查看内容占用，多次按可以切换不同的单位;
大写的E切换头部的显示单位
大写的P，下部可以按照cpu使用率排序；大写的M，下部可以按照内存使用率排序

  • 头部详解
    • 第一行：
      • 23:13:22表示当前系统时间
      • up 7 min表示系统启动了7分钟
      • load average分别表示过去 1 分钟、5 分钟和 15 分钟的系统负载平均值。负载值表示系统的繁忙程度，数字越高说明系统越繁忙。
    • Task表示任务信息： 96 total, 1 running, 95 sleeping, 0 stopped, 0 zombie：
      • 96 total：总任务数。
      • 1 running：当前正在运行的任务数。
      • 95 sleeping：当前处于休眠状态的任务数。
      • 0 stopped：当前被停止的任务数。
      • 0 zombie：僵尸进程的数量。
    • %Cpu(s):表示cpu使用情况： %Cpu(s): 0.0 us, 0.0 sy, 0.0 ni, 100.0 id, 0.0 wa, 0.0 hi, 0.0 si, 0.0 st：
      • 0.0 us：用户空间占用的 CPU 百分比。
      • 0.0 sy：系统空间占用的 CPU 百分比。
      • 0.0 ni：用户进程的优先级调整占用的 CPU 百分比。
      • 100.0 id：空闲 CPU 百分比。
      • 0.0 wa：等待 I/O 操作的 CPU 百分比。
      • 0.0 hi：硬中断占用的 CPU 百分比。
      • 0.0 si：软中断占用的 CPU 百分比。
      • 0.0 st：被虚拟机偷取的 CPU 百分比。
    • MiB Mem表示内存使用情况
    • MiB Swap:表示交换空间使用情况

在不指定运行环境的情况系，exec后面跟的指令会占用当前进程，和exec的系统调用差不多。

date

  • 自定义格式输出
    通过使用 + 后跟格式字符串，date 命令可以以指定的格式输出日期和时间。常用的格式选项包括：
    %Y：年份（4位数，如2024）
    %y：年份（2位数，如24）
    %m：月份（01-12）
    %d：日期（01-31）
    %H：小时（00-23）
    %I：小时（01-12，12小时制）
    %M：分钟（00-59）
    %S：秒（00-59）
    %p：AM 或 PM
    %A：星期几（全称，如Thursday）
    %a：星期几（简称，如Thu）
    %B：月份（全称，如August）
    %b：月份（简称，如Aug）
    例如： date +"%Y-%m-%d" 、date +"%I:%M:%S %p"
  • 日期计算
    date 命令可以通过 -d 或 --date 选项来进行日期的计算和操作。可以使用自然语言来指定日期偏移量。
    • date -d "tomorrow": 显示明天的日期
    • date -d "yesterday": 显示昨天的日期
    • date -d "+1 week": 显示一周后的日期
    • date -d "-1 month": 显示一个月前的日期
    • date -d "$(date +%Y-%m-01) +1 month": 显示下个月的第一天
  • 其他用法
    • date -d @1723118400 ： 将时间戳转换成可读日期
    • date -d "2024-08-08 12:00:00" +%s: 将指定的日期和时间转换为时间戳（从1970年1月1日起的秒数）
