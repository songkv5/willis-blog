特殊字段说明

  1. text
    text字段在match搜索时会被分词:在生成倒排索引以前，字符串会被分析器分成一个一个词项(貌似是keyword词项?). 不用于排序.

  2. keyword
    keyword类型适用于索引结构化的字段，比如email地址、主机名、状态码和标签。keyword类型的字段只能通过精确值搜索到

  3. date
    日期类型
    例子(支持三种日期类型):

  • yyyy-MM-dd HH:mm:ss,yyyy-MM-dd 日期格式的字符串
  • epoch_millis时间戳

  1. object
    对象类型.存储,json对象.

  2. range
    范围类型,细分为一下几种:integer_range、float_range、float_range、float_range、date_range
    例:

    expected_attendees的人数为10到20，时间是2015-10-31 12:00:00至2015-11-01。

  3. nested
    嵌套类型.一种特殊的object类型.

查询文档数量

  • 使用cat命令
    • http://host:port/_cat/count/indexName?v&pretty
  • 使用_count语法
    • http://host:port/indexName/_count
      cat api用法

查看索引信息

  • 查看主机上的全部索引
    • http://host:port/_cat/indices?v
      ?v表示显示出字段，没有-v只能看到字段值
  • 查看主机上某个索引
    • http://host:port/_cat/indices/nh_cp_house?v&pretty

需要注意的是，这个语法可以看到文档数量，这个文档数量和上面命令看到的文档数量不一样，这个文档数量包含了内嵌文档，所以值会更大

别名

查询语法

query语法规则：
query->[bool->must/should/must_not]match/term/terms/exist/missing/range。符合查询也可以进行嵌套

query 与filter对比

当使用于filter情况 时，查询被设置成一个“不评分”或者“过滤”查询。
当使用于query情况 时，查询就变成了一个“评分”的查询。和不评分的查询类似，也要去判断这个文档是否匹配，同时它还需要判断这个文档匹配的有 多好（匹配程度如何,从_score字段可以看出）。当指定其他排序字段和排序方式时,就没有评分匹配了

自 Elasticsearch 问世以来，查询与过滤（queries and filters）就独自成为 Elasticsearch 的组件。但从 Elasticsearch 2.0 开始，过滤（filters）已经从技术上被排除了，同时所有的查询（queries）拥有变成不评分查询的能力。
